# Lastenheft

## Inhaltsverzeichnis

1. Ausgangssituation und Zielsetzung
    1. Ausgangssituation: Daten über Stationen, Touren, Besucher etc. sollen zentral gespeichert werden.
    2. Ist-Zustand: Stift und Papier
    3. Aufgabenstellung: TDOT-App
    4. Beteiligte: Lehrer, Schüler
2. Anforderungen
    1. Funktionale Anforderungen: 
        1. Touren verwalten
        2. Stationen verwalten
        3. Besucher verwalten
        4. Schüler verwalten
        5. Daten analysieren
        6. [...]
    2. Nichtfunktionale Anforderungen:
        1. Qualitätsanforderungen: schnell, benutzerfreundlich, stabil, übersichtlich, Zugriffskontrolle, fehlerfrei
        2. Technische Anforderungen: MERN-Stack, Kubernetes, Load Balancing
3. Rahmenbedinungen
    1. Budget: (Wie viel insgesamt? Wie kommt dieser Preis zustande?, hier nicht anwendbar) 
    2. Zeitraum: 11.10.2023 - 04.04.2025
    3. Schnittstellen: Export/Import zu Excel, Anmelden über Azure
4. Abnahmekriterien
    * Stabile, effiziente, benutzerfreundliche App mit ausreichenden Zugriffskontrollmechanismen
    * Vollständige Dokumentation
    * Beseitigung aller Mängel
    * Erfüllung aller Anforderungen
