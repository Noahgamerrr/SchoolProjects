## 1) Projektziele

*Das Ziel des Projekts war es, eine vollständige WebApp für den Tag der offenen Tür in der HTL Villach zu erstellen. Diese WebApp sollte in der Lage sein, einen Großteil der Operativen und Management Arbeitslast des TDOT zu decken.*

| Ziel    | Status |
| -------- | ------- |
| Implementierung eines leicht zu verwendenen Frontends | Erfüllt    |
| Implementierung eines Datenschemas, das alle Aspekte des TDOT deckt.   | Erfüllt    |
| Implementierung von Tour Management | Erfüllt     |
| Deployment auf einen Live-Server | Erfüllt     |
| Verwalten von Stationsdaten | Erfüllt     |
| Verwalten von Besucherdaten | Erfüllt     |
| Verwalten & Einlesen von Schülerdaten | Erfüllt     |
| Gruppensystem für Guides und Station Workers | Erfüllt     |
| Bereitstellung von Management Tools (z.B. Notification System) | Erfüllt     |
| Deployment auf einen Live-Server | Erfüllt     |

Die Qualität der gelieferten Features ist im finalen Produkt laut Feedback der Stakeholder angemessen. Alle Nicht Funktionalen Anforderungen wurden genügend erfüllt.

## 4) Lessons Learned

- UI Features sollten von Anfang an auf kleinen Bildschirmen getestet werden, da Usability so früh erhöht werden kann und potentielle Revisions vermieden werden.
- Präsentationen sollten zentral von 1 Person geführt werden und sich durch die User Experience durchspielen.
- Präsentationen sollten resilient gestaltet sein, also zuvor schon mehrere Male getestet und durchgeführt werden um live Bugs zu vermeiden.



## 5) Empfehlungen für zukunftige Projekte

- Test-driven-development
- Klarere Analyse von Systemanforderungen
- Genaue Planung der Systemarchitektur -> Vermeidung eines wucherndes Projektes
